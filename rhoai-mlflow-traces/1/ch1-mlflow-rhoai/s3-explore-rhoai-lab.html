Lab: Explore the RHOAI Instance

Objective

Explore the predeployed Red Hat OpenShift AI instance to familiarize yourself with its configuration and with the new Gen AI Studio feature from 3.x releases.

Work In Progress

Before you begin

You need access to the predeployed Red Hat OpenShift AI (RHOAI) instance provided in your lab environment. Ensure you have:

  • Access credentials for the RHOAI dashboard

  • A web browser to access the RHOAI dashboard

Instructions

In this lab, you will log in to the RHOAI dashboard and explore its main features and components to understand the platform’s capabilities.

  1. Access the RHOAI Dashboard.

    1. Navigate to the RHOAI dashboard URL of your demo environment.

    2. Log in using the credentials provided for your demo environment.

  2. Explore the precreated project.

    Your RHOAI instance contains a model which is already deployed in a regular project named my-first-model — which is not an AI project.

    That model takes up all GPU capacity available to your RHOAI instance, so any change to the model deployment requires that you stop it and restart after making changes.

    1. On the left pane, click AI hub  Models and select the Deployments tab.

    2. On the Project combo box, type my and select my-first-model.

      Depending on the current page of the RHOAI daskboard, it may be necessary to first select All projects before you can search and select the my-first-model project.

    3. Verify that there is a model deployment named llama-32-3b-instruct and its status is Ready.

    4. Click View, under the Endpoints column for the model row, and verify that it provides only an internal endpoint. This means the model cannot be accessed from outside of its OpenShift cluster, unless you change its model deployment.

  3. Try the model with an Gen AI Studio Playground.

    1. On the left pane, click Gen AI studio  Playground.

    2. On the Project combo box, type my and select my-first-model.

      The project previously selected in one page of the RHOAI dashboard may not be the current selection in another page of the RHOAI dashboard.

    3. If you do not see a Configure pane, click menu:Settings. On the Configure pane, select the Model tab and verify that the selected model is named llama-32-3b-instruct. Ignore any prefix to the model name.

    4. On the right pane, click the Send a message…​ text and type a prompt, for example What is RHEL?

    5. You may get different answers, if you attempt the same prompt again, but they would all be correct and reasonable, hopefully.

  4. Verify that there are no RHOAI dasbboard pages related to MLFlow.

    1. On the left pane, expand the Gen AI studio cathegory, which contains entries named Playground and AI asset endpoints.

      After you enable MLFlow, there will be additional entries here.

    2. Click Gen AI studio  AI asset endpoints and select the MCP servers tab.

      Verify that there are no MCP servers configured for use by Gen AI studio playgrounds.

    3. On the left pane, expand the Develop & train cathegory, which contains entries named Feture store, Pipelines, Jobs, and Evaluations.

    4. On the top right, click the Applications icon, which is close to the Info (?) icon, and verify that there is not an entry named MLFlow.

After completing these steps, you have verified that you have a functional RHOAI instance with a pre-deployed model, which is available only for internal access, and that you can submit prompts to the model using an RHOAI playground.

In addition, you have also verified that MLFlow is not installed on your RHOAI instance.
