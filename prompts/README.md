The review-rules.md file was build as I wrote the content and did author reviews, as a running log of my decisions about formmatting and style, supplemented by a few generic styling rules;

The review-file.md contains a prompt to with an open adoc file, to review it. Claude does NOT follow it to the letter, for example: if I say "No" to one edit, it stops, instead of proceeding to the next remaining edit. I must tell it to continue in the claude chat window. :-(

Sometimes it hallucinates applying the rules, so I recommend, for now, reviewing and authorizing ech edit individually. This is easy to do from VSCode, using its Claude Code extension and its diff view.

Perform an author read and spellcheck run BEFORE using those prompts. It's a huge waste of tokens letting all typos and grammar to Claude. It can improve your writing, picking shorter and clearer sentences, breaking long ones, and rewording with active voice. But you must provide meaningful (and correct) content beforehand.
